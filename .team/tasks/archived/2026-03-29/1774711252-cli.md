## タスク
## 問題

テンプレート（master.md, conductor.md, common-header.md）がタスクファイルを直接操作しており、正規フローを経ずにタスクが作成・移動される。040のように draft のまま closed/ に直接コミットされるケースが発生。

## 修正内容

### 1. CLI サブコマンド追加

- `close-task --task-id NNN [--journal "summary text"]` — closed/ 移動 + Journal 追記
- `update-task --task-id NNN --status ready` — status 変更（sed 直接操作を廃止）

### 2. テンプレート修正

以下のテンプレートからタスクファイル直接操作を CLI 呼び出しに置き換え:

- **master.md**: `sed -i '' 's/status: draft/status: ready/'` → `bun run main.ts update-task --task-id NNN --status ready`
- **conductor.md**: `mv $TASK_FILE .team/tasks/closed/` → `bun run main.ts close-task --task-id NNN`
- **common-header.md**: `.team/tasks/open/ にファイル作成` → `bun run main.ts create-task` を案内
