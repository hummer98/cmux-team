## タスク
## 目的

cmux-team の配布方法を Claude Code Plugin 単体から npm パッケージ + Plugin 併用に移行する。
`cmux-team start` でどのプロジェクトからでもシェル起動できるようにする。

## 背景

- 現状: Plugin として配布。CLI 起動には plugin cache 内の main.ts への複雑なパス指定が必要
- 問題: Claude Code セッション外からの起動が困難。`claude '/cmux-team:start'` は surface レイアウトが崩れる
- 解決: npm パッケージとして配布し、`cmux-team` CLI コマンドを PATH に追加

## サブタスク

### 1. npm パッケージ構造の構築
- ルートに package.json を追加（name: cmux-team, bin: { cmux-team: ./bin/cmux-team.js }）
- bin/cmux-team.js — 薄いエントリポイント（main.ts を bun run するラッパー）
- postinstall スクリプトで:
  - manager/ の依存解決（bun install）
  - `claude plugin add hummer98/cmux-team` を実行（スキル/コマンドの Claude Code 連携）
- .npmignore で不要ファイルを除外

### 2. CLI サブコマンドの整備
- `cmux-team start` — daemon 起動（現在の main.ts start と同等）
- `cmux-team stop` — daemon 停止
- `cmux-team status` — ステータス表示
- `cmux-team create-task` 等 — 既存サブコマンドをそのまま透過
- main.ts のパス解決: npm インストール先から相対パスで自動解決

### 3. 起動フローの更新
- /cmux-team:start コマンド（commands/start.md）を更新:
  - npm インストール済みなら `cmux-team start` を呼ぶ
  - フォールバック: 従来の plugin cache 検索
- spawn-team.sh の更新（必要に応じて）
- Conductor テンプレート内の main.ts パス解決を更新

### 4. ドキュメント修正
- README.md / README.ja.md — インストール手順を npm 中心に書き換え
- CLAUDE.md — リポジトリ構造・インストール方法セクション更新
- install.sh — 非推奨化（npm を推奨、レガシー向けに残す）

### 5. バージョン同期
- npm バージョンと plugin.json バージョンを一致させる仕組み
- リリースフロー（/release）の更新: npm publish を追加

## 注意
- 既存の Plugin インストール（claude plugin add）は引き続きサポート
- npm が primary、Plugin-only が fallback
- cmux と Claude Code が前提なので、postinstall で claude plugin add を直接実行してよい
